# Nextwork PRD

## 1. 产品一句话

Nextwork 是一个面向年轻人的数据决策支持系统：先帮助他们知道自己是谁，再用持续更新的社会需求数据告诉他们世界需要什么，最后给出可解释、可验证、可行动的升学与职业路径。

## 2. 产品愿景

让数据开口，照亮前途。

年轻人并不缺信息。他们缺的是能被整理、解释、对照和转化成行动的信息。专业目录、招聘岗位、政策文件、创业新闻、融资消息、开源趋势、竞赛项目和学校培养方案都在说话，但普通学生和家长很难听懂。

Nextwork 要做的是把这些信号翻译成人能理解的路径：

- 我是谁
- 社会需要什么
- 我现在和目标之间差什么
- 我下一步该学什么、做什么、验证什么

## 3. 问题背景

### 3.1 往日分类正在失效

过去的职业规划依赖静态分类：

- 专业分类
- 行业分类
- 岗位分类
- 文理分类
- 兴趣测试分类

这些分类仍有用，但不足以应对当下的变化。AI Agent、具身智能、低空经济、AI 医疗、新能源、AIGC 内容和企业自动化都不是单一专业能覆盖的领域。一个岗位可能同时需要工程、产品、数据、行业知识和表达能力。

### 3.2 年轻人真正需要的是动态对照

年轻人需要看到：

- 当前社会到底在招聘什么能力
- 哪些能力是所有方向都需要的基础能力
- 哪些能力只适合特定赛道
- 哪些方向只是概念热，哪些方向已经有岗位和预算
- 自己当前画像与这些机会之间的距离
- 应该选什么专业群，而不是只押一个专业名

## 4. 用户

### 4.1 核心用户

- 高考生：想根据未来就业和创业机会选择专业。
- 大学生：想判断是否转专业、辅修、实习或补技能。
- 应届生：想知道自己适合哪些岗位，缺什么能力。
- 早期创业者：想看社会需求和人才结构，判断赛道是否值得做。

### 4.2 影响用户

- 家长
- 高中老师
- 大学就业指导老师
- 职业规划咨询师
- 教育机构内容团队

## 5. 核心理念

### 5.1 先知道自己是谁

用户画像不是 MBTI 式标签，而是可用于决策的变量：

- 学科优势：数学、物理、生物、语文、英语、艺术表达。
- 能力倾向：编程、硬件动手、沟通组织、审美表达、商业敏感、研究耐心。
- 约束条件：城市、家庭预算、升学目标、风险承受能力、是否接受长期深耕。
- 行动证据：作品、竞赛、项目、实习、开源、社群贡献。

### 5.2 再知道社会需要什么

社会需求不是抽象口号，而是信号流：

- 企业在招什么岗位
- 岗位要求什么技能
- 创业赛道最终需要什么人
- 政策正在推动什么方向
- 新增专业说明什么供给变化
- 哪些技能在跨岗位重复出现
- 哪些信号正在上升、衰减或分化

### 5.3 最后形成路径

输出不应该只是“你适合人工智能专业”，而应该是：

- 首选专业群
- 备选专业群
- 目标岗位方向
- 关联创业赛道
- 大学四年学习路线
- 必做作品项目
- 风险提示
- 下一步验证任务

## 6. AI HOT 信息机制迁移

AI HOT 的价值不在于它只是一个资讯站，而在于它把“混乱信息”变成了“可消费的信息机制”。Nextwork 迁移这个机制，但换成职业与升学场景。

### 6.1 多信源采集

职业信号来源：

- 大厂和成长型公司招聘官网
- 校招和实习岗位
- 政策文件与政府报告
- 本科专业目录和新增专业
- 创业融资、产品发布、竞赛方向
- 开源项目、论文、技术社区趋势
- 学校培养方案和课程地图

### 6.2 AI 降噪和归并

系统需要识别：

- 同一事件被多个渠道重复报道
- 同一个岗位换名字重复发布
- 不同公司对同一能力的不同表述
- 一个赛道的概念热度和真实招聘需求是否匹配

输出应归并为“职业信号”，而不是原始信息堆叠。

### 6.3 精选 / 全量双层

- 精选：给普通学生和家长看的高价值职业信号。
- 全量：给研究者、咨询师、老师和 Agent 分析用。

默认展示精选，避免信息过载。只有用户明确要“全部”“完整”“原始列表”时，才进入全量视图。

### 6.4 日报 / 周报 / 阶段报告

Nextwork 可生成：

- 职业信号日报：今天哪些岗位、政策、赛道发生变化。
- 专业选择周报：本周哪些专业群关联信号增强。
- 个人路径报告：某个学生画像与社会需求的匹配和缺口。

### 6.5 推荐理由

每条信号都必须有推荐理由：

- 为什么这条信息重要
- 它影响哪些专业群
- 它说明哪些能力在变重要
- 它适合哪类学生
- 它有什么风险或噪声

### 6.6 Agent / API / RSS 接入

后续应该开放：

- RSS：订阅职业信号精选、专业群周报、赛道变化。
- REST API：按专业、能力、赛道、时间窗、关键词查询。
- Agent Skill：让 ClaudeCode、Codex、Cursor 等 Agent 直接查询职业信号。

示例自然语言：

- 最近 AI 医疗方向需要什么能力？
- 低空经济相关专业怎么选？
- 数学强但不想纯写代码的学生适合什么方向？
- 最近哪些创业赛道开始真正招人？
- 计算机专业还值得报吗，风险在哪里？

## 7. 产品结构

### 7.1 小程序端

首页默认路径：

1. 学生画像选择
2. 首选专业群
3. 推荐理由
4. 社会需求信号
5. 创业赛道拆解
6. 大学四年路线
7. 风险提示
8. 下一步验证任务

### 7.2 数据后台

后台需要支持：

- 信源管理
- 抓取任务状态
- 职业信号归并
- 标签体系管理
- AI 评分和人工校正
- 推荐理由审核
- 更新日志

### 7.3 Agent 接入

Agent 侧重点不是展示页面，而是回答问题和生成报告：

- 给一个学生画像，生成专业建议。
- 给一个专业群，解释就业与创业信号。
- 给一个岗位方向，拆能力栈和学习路线。
- 给一个城市或学校，做本地化路径建议。

- 专业群推荐
- 大学四年路线

### 9.2 下一阶段

- 新增 `CareerSignal` 数据模型
- 把岗位、赛道、政策、专业目录统一成信号流

### 9.3 暂不做

- 不做最终录取概率预测
- 不做学校排名推荐
- 不做收费咨询闭环
- 不做不可解释的黑箱测评
- 不用单一问卷决定用户命运

## 10. 体验原则

- 结论先行：先给用户看推荐，再展开依据。
- 证据可追溯：所有社会需求信号必须能回到原始来源。
- 语言要像人话：不要把 API 参数、内部标签、模型术语直接扔给学生。
- 风险要透明：热门方向必须配风险说明。
- 推荐要可行动：每次输出都要有下一步验证任务。

## 11. 验收标准

### 11.1 用户价值

- 用户能在 1 分钟内理解产品在做什么。
- 用户能在 3 分钟内得到一个专业群建议。
- 用户能看到推荐背后的社会需求证据。
- 用户能知道下一步要验证什么，而不是只得到一个结论。

### 11.2 数据质量

- 每条职业信号有来源、时间、摘要、标签、评分和推荐理由。
- 同一事件不会在精选页重复刷屏。
- 精选信号能覆盖岗位、政策、赛道、专业目录四类核心输入。
- 推荐结果能解释“适合谁”和“不适合谁”。

### 11.3 工程质量

- `npm run build` 通过。
- 数据模型和 UI 解耦。
- 种子数据可替换为真实 adapter 输出。
- 未来 API / Agent 接入不需要重写业务模型。
